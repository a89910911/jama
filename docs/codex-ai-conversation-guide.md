# Codex AI 助手对话指南

## 工作方式

Codex AI 助手会先判断本轮是“咨询讨论”还是“执行项目任务”：

1. 明确的常用表达会直接路由到对应项目能力。
2. 未命中固定表达时，Codex 会结合当前项目、当前剧集和最近对话生成结构化执行计划。
3. 只有高置信度且明确要求生成、修改、提取或保存时，才会调用项目写库逻辑。
4. 文字结果由项目服务校验后写入数据库；图片使用 Codex 原生图片能力生成，再由项目服务保存并绑定。
5. 咨询、分析、评价和状态询问不会修改数据库。

## 支持的项目能力

| 目标 | 执行动作 | 入库结果 |
| --- | --- | --- |
| 生成剧本 | 生成当前集或项目剧本 | `episodes` |
| 改写本集 | 覆盖当前剧集内容 | `episodes` |
| 续写本集 | 追加到当前剧本末尾 | `episodes` |
| 提取资源 | 提取角色、道具、场景说明及图片提示词 | `characters`、`props`、`scenes` 及剧集关联 |
| 生成资源图 | 每个资源生成一张独立图片 | 资源表、`image_generations`、`assets` |
| 优化资源提示词 | 优化指定角色、道具或场景的图生提示词，不生成图片 | 角色 `polished_prompt`、道具 `prompt`、场景 `polished_prompt` |
| 生成全部分镜 | 生成结构化分镜及角色、道具、场景关联 | `storyboards` 及关联表 |
| 补充分镜说明 | 补充标题、画面说明、布局、动作、结果或氛围 | `storyboards` 对应字段 |
| 优化分镜提示词 | 优化原始、通用图片或视频提示词，不生成图片 | `storyboards.image_prompt`、`polished_prompt`、`video_prompt` |
| 生成分镜图 | 每条分镜生成一张独立首帧 | `storyboards`、`image_generations`、`assets` |
| 生成单图 | 生成一张普通项目素材图 | `image_generations`、`assets` |

## 推荐表达

- 说明动作：生成、改写、续写、提取、补齐、重新生成。
- 说明范围：当前集、全部、第 3 个分镜、角色凯尔、尚未有图的场景。
- 说明写入要求：保存到当前集、绑定到对应资源、写入素材库。
- 说明覆盖策略：需要替换旧数据时明确说“重新生成并覆盖”；未说明时，已有图片默认跳过。
- 说明图片类型：资源图、分镜首帧或普通单图不能混用。

示例：

```text
请根据这个创意生成当前集完整剧本并保存：少女在森林里遇见会说话的狐狸。
```

```text
从当前剧本提取角色和道具，补充外观说明与图片提示词并写入资源库。
```

```text
给尚未有图片的角色、道具和场景分别生成独立资源图，并绑定到对应资源。
```

```text
把当前集完整拆成所有分镜，补齐动作、结果、镜头说明和画面布局。
```

```text
第二镜还是空的，给它配一张独立电影首帧并挂回那条分镜。
```

## 常见意图的推荐说法

| 你的意图 | 推荐表达 |
| --- | --- |
| 生成凯尔角色图 | `为角色凯尔生成一张独立角色图，并绑定到凯尔的资源记录。` |
| 不满意，重新生成凯尔角色图 | `我不满意刚才的凯尔角色图，请重新生成一张并替换凯尔当前图片。` |
| 只优化凯尔图生提示词 | `优化角色凯尔的图生提示词并保存，不要生成图片。` |
| 给剧本生成全部分镜图 | `为当前集所有分镜分别生成独立首帧图片，并绑定到对应分镜。` |
| 重生成第 2 个分镜图 | `重新生成第 2 个分镜的独立首帧，并覆盖当前分镜图。` |
| 补充全部分镜说明 | `补充当前集所有分镜的画面说明和空间布局说明并保存。` |
| 补充第 2 个分镜说明 | `补充第 2 个分镜的画面说明、动作结果和空间布局并保存。` |
| 生成第 2 镜通用优化提示词 | `生成第 2 个分镜的通用优化提示词，写入通用优化提示词字段，不要生成图片。` |
| 优化第 2 镜视频提示词 | `优化第 2 个分镜的视频提示词，保持首帧、动作和运镜连续，并写入视频提示词字段。` |
| 优化第 2 镜全部提示词编辑字段 | `优化第 2 个分镜提示词编辑区的原始提示词、通用优化提示词和视频提示词，并分别写入对应字段。` |

“生成/重新生成图片”和“优化提示词”是不同操作：前者会调用 Codex 原生图片能力并产生图片，后者只生成文字并更新提示词字段。

## 需要注意

- 复杂流程建议依次发送：剧本 → 资源说明 → 资源图片 → 分镜 → 分镜图片。这样更容易检查结果并重试失败部分。
- 当前页面选中的剧集决定“本集”“这一镜”等表达指向哪里。
- 指定单个资源或分镜时请给出名称或编号，避免执行范围扩大。
- “所有图片”默认只补缺失图片；使用“重新生成并覆盖所有图片”才会替换已有图片。
- 图片会逐项生成，批量任务耗时较长；可以停止后再次发送相同要求，系统会跳过已完成项。
- 只有助手明确回复“已写入”“已保存”或“已绑定”时，才表示项目入库已经成功。
- 如果只是咨询，请使用“分析、解释、建议、为什么”等表达；咨询回复不会写库。
- 快捷按钮只负责填入示例。编辑示例后，系统会重新理解修改后的自然语言，不会强制沿用按钮原意。
